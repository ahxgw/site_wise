Context

xvideos.com is one of the largest adult video sites — a free, ad-funded tube mixing uploader UGC with studio catalogs. It is owned by WGCZ Holding (WebGroup Czech Republic, Prague), which also runs the sister site xnxx.com. Retrieval Notes (site-specific deltas)

Only where xvideos.com differs from the default platform assumptions.

Identifier & dedup

A video carries three IDs: an internal numeric id_video (e.g. 90312082), a per-listing obfuscated token (opitdvi9428), and a title slug. The canonical og:url (token + slug) is authoritative; several URL forms resolve to one video, so dedup on og:url or numeric id_video. Much of the catalog is mirrored on sister site xnxx.com under different IDs — cross-site perceptual hashing is needed to collapse duplicates.

Retrieval unit

The video is the atom. Pornstar / channel / tag / category pages are navigational hubs and the join graph, not retrieval targets — but they are the only thing the sitemaps enumerate, so they bootstrap discovery.

Cross-modal joins

The media UUID in the CDN path (…-cdn.com/<uuid>/…) is shared between thumbnailUrl and contentUrl, keying image↔video. On-page links join video → pornstar → channel → tags. No stable ISRC/UPC-style external key exists.

Authoritative content field

Titles and descriptions are uploader-authored and keyword-stuffed for internal/Google search — the least trustworthy fields. The structured fields (duration, uploadDate, IDs) are clean machine output; tags are the most reliable topical signal.

Freshness & recrawl

Video metadata is immutable after upload, so it is crawl-once. But the contentUrl token expires, so the page must be re-fetched to obtain a fresh signed media URL. Trending / browse / front pages churn constantly (changefreq: always).

Quality & spam

Beyond ordinary SEO spam and re-uploads, the dominant concern is content legality (CSAM, non-consensual material). The trust layer is regulatory — age verification and DSA-mandated moderation — not an exposed quality score. Any index must classify-and-gate before storage.

Internal vs external search

Unlike walled platforms, xvideos is built for external search: open robots, entity sitemaps, slugged URLs, and full JSON-LD. site:xvideos.com is deep, and Google is a real discovery channel — the inverse of open.spotify.com, whose catalog is deliberately hidden from crawlers.

User query profile

Category / tag queries, performer-name lookups, and long-tail descriptive phrases. High navigational and browse intent; little informational text retrieval.

Indexing Plan

  • Retrieval unit: the video, keyed by numeric id_video; pornstar / channel / tag / category as entity records.
  • Primary keys & joins: id_video as PK; obfuscated token + slug as URL aliases (canonical = og:url); media UUID joins thumbnail↔contentUrl; entity IDs from the per-entity sitemaps.
  • Ingestion paths, ranked: 1) entity sitemaps — no auth, but they enumerate categories/tags/performers, not videos. 2) per-page JSON-LD/OG — no auth, complete video metadata including the signed contentUrl. 3) internal browse/pagination crawl — the only way to enumerate the video corpus (sitemaps don't), respecting the /video-vote/ disallow. 4) /embedframe/{id} — no-auth display surface. No official API exists.
  • Recrawl cadence: trending/browse pages daily; video metadata crawl-once (immutable); re-fetch a page only to refresh an expired contentUrl token; entity sitemaps weekly.
  • Derivatives to generate: ASR transcripts and keyframe sampling are technically possible here (the MP4 is fetchable, unlike Spotify) — but only within reuse rights; perceptual video hashing for cross-site (xnxx) dedup; and mandatory age/CSAM classification before any retention.
  • Demo to build: a JSON-LD video-record harvester — given a video URL, emit the index doc (IDs, tags, duration, uploadDate, thumbnail, signed contentUrl) and flag the token's TTL, demonstrating that the corpus is technically open while the real gate is legal, not technical.

Risks to the Index

  • The wall is legal, not technical. As a DSA VLOP (designated Dec 20 2023) and a target of age-verification mandates (EU, UK Online Safety Act, US state laws), an anonymous crawler increasingly meets an age-gate, not content. inferred Age verification is becoming the access gate. Basis: the CSP loads yoti: and global.frcapi.com facial-age sources (Probe Log).
  • Content-legality liability: indexing or redistributing risks CSAM and non-consensual material; classification + takedown handling is mandatory, not optional.
  • Signed-URL rot: contentUrl tokens expire, so any cached media link decays; the page must be re-derived for a fresh URL.
  • No contract to rely on: there is no API or licensing program — access depends on robots/markup that the owner can change unilaterally, and sister-site (xnxx.com) duplication inflates the corpus.
