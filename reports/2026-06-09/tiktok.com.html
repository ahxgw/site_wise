Context

TikTok is ByteDance's global short-video app, monetized by ads and TikTok Shop commerce. Since January 23, 2026 its US operation sits in a separate company, TikTok USDS Joint Venture LLC (ownership in Verified Metrics), which controls US user data, the US recommendation algorithm (run in Oracle's cloud), and US moderation. For an index builder the unit is the video post: a 19-digit ID under an @handle, joined to a sound, a creator, hashtags, and comments. TikTok exposes this corpus three ways, none complete: public web pages embedding a full JSON state blob (served unreliably — see Probe Log), a keyless oEmbed endpoint, and a Research API gated to qualified academic and nonprofit researchers. There is no sitemap of content and no feed. inferred The public-web SEO surface is deliberately minimal. Basis: the only sitemap holds 5,660 corporate/legal URLs and zero videos or profiles; fetched pages carry no JSON-LD or OG tags; robots.txt allows generic crawlers only a short path whitelist (Probe Log, 2026-06-10).

Retrieval Notes (site-specific deltas)

Only where TikTok differs from the default platform assumptions.

Identifier & dedup

Video ID is a 64-bit integer whose top 32 bits are the Unix creation time — IDs sort chronologically for free. inferred Basis: sample video 6718335390845095173 >> 32 = 1564234353, five seconds from its page-state createTime 1564234358 (Probe Log). Handles are mutable; user id and secUid are the stable creator keys. vm.tiktok.com share links are opaque aliases that must be resolved before dedup.

Retrieval unit

The video post, carrying caption, creator, music ID, hashtags, counts, AIGC flags. For trend queries the sound is the better unit: one music ID clusters thousands of videos into a meme lineage no text field captures.

Cross-modal joins

All joins ride three stable keys: video ID, music ID, secUid. Hashtags join via /tag/{name} but are free-text and squattable. Shop product IDs exist in video records but their pages are robots-disallowed — the commerce join is visible only in-app.

Authoritative content field

Captions are sparse and emoji-laden; the video itself carries the meaning. subtitleInfos in page state holds creator/auto captions when present (zero tracks on the sampled video — coverage is an open question). ASR is not optional here; it is the primary text field.

Freshness & recrawl

Because IDs encode creation time, "everything since timestamp T" is an ID-range scan — the Research API supports date-bounded queries, making incremental ingestion unusually clean for a platform with no feeds.

Quality & spam

Page state ships integrity labels worth ingesting: IsAigc/AIGCDescription (AI-generated flags) and indexEnabled — a per-video flag for whether the creator permits external indexing. An index should honor indexEnabled: false as an opt-out signal.

Internal vs external search

TikTok search is a destination for how-to/product/local queries, and the platform keeps external engines at arm's length: /search? is robots-disallowed, so query-level content is invisible to crawlers. Google holds video pages only because generic agents stay allowed.

User query profile

Trend, sound, creator, and "what was that video" queries — many better answered by audio/visual match (music ID, frame embedding) than by caption text.

Indexing Plan

  • Retrieval unit: video post; sounds as a first-class secondary unit clustering posts by music ID.
  • Primary keys & joins: video ID (time-ordered, see Retrieval Notes); creator = secUid; sound = music ID; resolve vm.tiktok.com aliases at ingest.
  • Ingestion paths, ranked: 1) Research API — 100K records/day of videos+comments with date-bounded queries, but gated to academic/nonprofit status and revocable. 2) oEmbed + page-state JSON — keyless metadata for seed sets; page state is rich but served on ~1 of 4 attempts (Probe Log), so plan for retries. 3) Embed player — display layer, zero metadata beyond oEmbed. 4) Commercial Content API — the ads corpus, EU coverage only.
  • Recrawl cadence: new posts via Research API date queries daily; engagement counts on trending posts hourly via page state; back-catalog counts monthly; comment threads only for posts above an engagement floor.
  • Derivatives to generate: ASR on every indexed video (captions are not the content); frame OCR for text-on-screen memes; audio fingerprints to join re-uploads to the same sound; re-fetch thumbnails on access since CDN URLs expire.
  • Demo to build: an oEmbed enrichment loop — paste any TikTok URL (including vm. short links), resolve, pull the 15 oEmbed fields plus page-state stats when the fetch wins, and emit one index document showing which fields came keyless vs. gated.

Risks to the Index

  • Jurisdiction split: since Jan 23 2026 the US algorithm and US user data sit in TikTok USDS JV (Oracle cloud) while the rest of the world stays ByteDance (announcement). Two operators can mean diverging corpora, policies, and API terms by region.
  • Research API is a privilege, not infrastructure: eligibility is region- and status-gated, quotas are fixed ("we can't grant exceptions" — FAQ), and access is revocable; an index must not make it the only ingestion path.
  • Page-state dependence is fragile: the embedded JSON blob is an internal hydration format, already bot-walled intermittently (Probe Log); a rename or full wall breaks path 2 overnight.
  • inferred Scraper enforcement will keep hardening. Basis: robots.txt blocks 25 named crawlers including ByteDance's own Bytespider (observed 2026-06-10) — a company that bans its own bot from its consumer site treats anti-scraping as policy, not accident.
  • EU pressure opens doors: DSA proceedings produced binding ad-transparency commitments (EC, 2026) — researcher access is likeliest to expand in the EU.
