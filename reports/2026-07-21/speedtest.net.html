Context

Speedtest.net is Ookla's free browser/app tool for measuring download/upload speed and latency, founded 2006 by Mike Apgar and Doug Suttles. Ziff Davis acquired Ookla in 2014; Ziff Davis then sold its whole Connectivity division (Ookla/Speedtest, Downdetector, Ekahau, RootMetrics) to Accenture for $1.2B, announced 2026-03-03 and closed 2026-06-17. Monetization runs three tracks: the free consumer test surfaces ISP affiliate offers on "Performance Directory" city pages; the enterprise product Speedtest Intelligence licenses the same crowdsourced measurement stream to ISPs, carriers, and governments; and Ookla Open Data publishes a de-identified quarterly extract under CC BY-NC-SA as goodwill/marketing. For an index builder, the site itself carries little classic web content — no blog feed, almost no media — but three distinct structured layers: a live public test-server directory (16K+ geocoded nodes), US-only city pages with dated median-speed stats and ISP offer entities, and a monthly global country ranking. The real value sits in Ookla's underlying row-level measurement dataset, which the public site never exposes directly.

Retrieval Notes (site-specific deltas)

Only where speedtest.net differs from the default platform assumptions.

Identifier & dedup

Apex speedtest.net (Fastly) 301s to www.speedtest.net (Cloudflare) — canonicalize on www. Performance Directory pages key on a /performance/{country}/{state}/{city} slug path; the server directory uses a separate numeric id. No single ID space spans both.

Retrieval unit

Not the page — the underlying speed-test measurement (client geo, ISP, download/upload/latency, timestamp). The public site never exposes row-level records; only aggregated derivatives (per-city median, per-country rank, per-tile Open Data raster) are retrievable without a paid Speedtest Intelligence contract.

Cross-modal joins

inferred ISP identity is the only practical join key across the server directory, Performance Directory, and Open Data, and it is not a formal one. Basis: sampled server-list rows carry a numeric isp_id while Performance Directory JSON-LD carries only a free-text brand.name, with no shared identifier observed between the two payloads (Probe Log).

Freshness & recrawl

Three different cadences layered on one product: server directory changes continuously, Performance Directory pages update monthly (per-page dateModified), Open Data refreshes quarterly. Recrawl scheduling should key off object class, not a single site-wide cadence.

Internal vs external search

Performance Directory city-granularity coverage (ISP affiliate monetization) is US-only (Probe Log); the global-scope product is the country-level Global Index, a different granularity and a different robots.txt exclusion group. A query for a non-US city has no matching Performance Directory page to retrieve.

Indexing Plan

  • Retrieval unit: one Performance Directory city-record (country/state/city, median mobile+fixed download/upload/latency, ISP offer list, dateModified) plus one Open Data tile-record (zoom-16 tile, quarter, technology, download/upload/latency).
  • Primary keys & joins: city slug path as the Performance Directory key; tile geometry + quarter as the Open Data key; ISP name as a soft, non-canonical join across the two (Retrieval Notes).
  • Ingestion paths, ranked: 1) Ookla Open Data (aws s3 --no-sign-request s3://ookla-open-data) — the only true bulk path, free, CC BY-NC-SA 4.0, quarterly, no auth. 2) Performance Directory sitemap crawl + JSON-LD scrape — public GET, but US-only coverage. 3) servers.json — unauth but proximity-scoped, no documented full enumeration — open. 4) Speedtest Intelligence — the row-level commercial dataset, paid/contract-gated, out of scope for open indexing.
  • Recrawl cadence: Open Data quarterly (matches publisher cadence); Performance Directory monthly (matches observed dateModified deltas and Global Index refresh); server directory not worth recrawling as a content corpus.
  • Derivatives to generate: none media-side — this is a numeric dataset, not media. The one derivative worth building is a per-city time series by diffing successive Open Data quarters.
  • Demo to build: pull one Open Data quarterly Parquet tile (no-sign-request S3 GET) covering San Diego and diff its median download figure against the same city's Performance Directory JSON-LD figure — a concrete, no-auth cross-validation of Ookla's bulk vs. page-rendered numbers.

Risks to the Index

  • Ownership transition: Ookla moved from Ziff Davis to Accenture on 2026-06-17 (Verified Metrics); Accenture's stated enterprise "network intelligence" strategy could deprioritize the free consumer tool and public Open Data/Global Index if they stop serving the new business model.
  • No formal ISP join key: cross-referencing the server directory, Performance Directory, and Open Data by ISP currently requires fuzzy name matching (Retrieval Notes) — an ISP rename or brand consolidation would silently break joins.
  • Open Data is non-commercial-licensed: CC BY-NC-SA 4.0 means an ad-supported or otherwise commercial index cannot ingest it without a separate license from Ookla.
  • No documented server-directory enumeration: servers.json is proximity-scoped with no bulk-list endpoint (Probe Log); crawling it fully would require synthesizing lat/lon query grids, which is fragile and could be rate-limited without notice.
