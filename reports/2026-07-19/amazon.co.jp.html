Context

Amazon.co.jp is Amazon's Japanese-language storefront, launched Oct 31, 2000 with a books catalog for Japan and Japanese speakers worldwide, and operated today by subsidiary Amazon Japan G.K. Japan is Amazon's fourth-largest disclosed geography: $30.688B net sales in FY2025, 4.3% of Amazon's $716.9B consolidated total, folded into the International segment rather than reported as its own line. The site runs the same retail-margin + marketplace-commission + advertising stack as amazon.com, plus a Prime program (launched Jun 8, 2007 at ¥3,900/yr, now ¥5,900/yr since Aug 2023) bundling delivery, video, and music. For an index builder the most consequential fact isn't Japan-specific: on May 15, 2026 Amazon retired the Product Advertising API worldwide, including its Japan locale, in favor of an OAuth-based Creators API with a new eligibility bar. inferred Any indexer still pointed at PA-API for amazon.co.jp has been silently dark for over two months as of this report. Basis: PA-API's own Japan locale doc (fetched 2026-07-19) states "PA-API will be deprecated on May 15th, 2026" with no grace-period language, and the doc banner says the site "is no longer maintained."

Retrieval Notes (site-specific deltas)

Only where amazon.co.jp differs from the default platform assumptions and from the amazon.com report's baseline.

Identifier & dedup

ASIN mechanics are identical to amazon.com, but the JP marketplace (www.amazon.co.jp, per the PA-API/Creators locale doc) is its own ASIN namespace — a product sold in both the US and Japan carries two unrelated ASINs, joinable only through GTIN/brand matching, not the ASIN itself.

Authoritative content field

inferred Encoding/locale metadata is handled slightly differently than on amazon.com. Basis: the captcha interstitial served to this probe declares lang="jp" (nonstandard; BCP 47 calls for ja) and charset="UTF-8", observed 2026-07-19 on both the homepage and a product-page fetch.

Freshness & recrawl

The dominant freshness event this year isn't a catalog change, it's a pipe change: the PA-API path every third-party JP indexer relied on went dark May 15, 2026 (Verified Metrics). Any recrawl plan built before that date needs its ingestion layer rewritten, not just re-run.

Indexing Plan

  • Retrieval unit: product = JP-marketplace ASIN variation family; offers and rating summaries attached as fast-refresh sub-records (same shape as amazon.com).
  • Primary keys & joins: marketplace (www.amazon.co.jp) + ASIN; offer = ASIN + sellerId + condition; GTIN/brand for cross-marketplace joins to amazon.com's own ASIN namespace.
  • Ingestion paths, ranked: 1) Creators API — the only documented forward path for catalog/images/offers/browse nodes; gated behind an approved creators account (eligibility detail: open). 2) SP-API — sellers' own catalog plus ANY_OFFER_CHANGED push for price/stock churn. 3) Public crawl — captcha-walled for both homepage and product pages (Probe Log); not viable. 4) Review text — no legitimate path at any point in PA-API's or Creators API's history.
  • Recrawl cadence: offers via SP-API push, not polling; catalog metadata weekly; browse-node tree monthly.
  • Derivatives to generate: GTIN concordance across the amazon.co.jp and amazon.com ASIN namespaces; Japanese text normalization (kana/kanji/romaji variants) for query matching against titles.
  • Demo to build: a Creators API probe script — attempt the retired PA-API GetItems call against the JP marketplace to confirm it fails, then walk the OAuth2 Creators API registration flow and document exactly what a fresh account can and can't fetch before any sales accrue.

Risks to the Index

  • Hard cutover, no bridge: PA-API's May 15, 2026 retirement is an authentication and request-shape rewrite, not a config change (OAuth2 replaces AWS Signature) — an index built on the old client library breaks completely, not gradually.
  • inferred Cold-start got harder, not easier. Basis: the retired PA-API granted a 60-day, 8,640-req/day trial with zero sales required (Verified Metrics history in the amazon.com report); third-party Creators API migration guides describe a 10-qualifying-sale bar instead — a new indexer with no existing traffic may not be able to start at all pending primary-source confirmation.
  • Antitrust exposure on the offer graph: the JFTC's Nov 2024 on-site inspection targets Buy Box/seller-pricing practices (Verified Metrics); a forced change to Buy Box logic would change what "the offer" means for JP ASINs specifically.
  • Bot defense is still first-party infrastructure: the validateCaptcha wall (Probe Log) blocks generic crawl identically to amazon.com's WAF challenge, just via a different code path.
