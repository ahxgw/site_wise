Search and Retrieval Perspective

How would you treat this site if you were indexing it, building agent tools that call it, or training on its content? This is the lens that distinguishes "high-traffic site" from "information source".

Identifier & canonical URL

Stable IDs, URL aliases that resolve to the same resource, dedup rules.

Crawl & access policy

robots.txt, paywall / login wall / geo wall / age wall, bot defenses, llms.txt or AI training policy.

Official APIs & feeds

API surface, quota, RSS / Atom / sitemap, OpenSearch, oEmbed, JSON-LD / microdata / OG.

Retrieval unit (what is a "document"?)

Page, post, transcript segment, message — which granularity is actually useful to retrieve.

Authoritative content field

When title / description / body / transcript / OCR disagree, which one wins? Where does SEO distortion live?

Freshness & decay

Mutability, deletion rate, decay half-life, recrawl cadence implications.

Quality signals & spam patterns

Official trust tiers (verified / authority), known spam / farm patterns, manipulation incentives.

Licensing & reuse

What can a third-party search UI legally show? Snippets, thumbnails, embeds, transcripts.

Personalization & non-determinism

Logged-out vs logged-in vs cookied differences; whether anonymous crawl reflects real "popular".

Region / language splits

Geo / language gating, parameters (gl, hl), copyright-territory differences.

Internal search vs external

How good is on-site search? Do users escape to Google with site:? Implication for retrieval strategy.

Outbound vs inbound posture

Hub or sink? Where it sits in the broader knowledge graph; embed footprint elsewhere.

Anti-manipulation / ranking surface

If someone wanted to be found here, what gates them in? Spam defenses, ranking factors, reverse-SEO risk.

User query profile

What does a typical query on this site look like? Which retrieval intents fit, which do not.
