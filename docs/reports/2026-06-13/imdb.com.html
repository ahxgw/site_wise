Context

IMDb is the world's reference database for movies, TV, and the people who make them — an Amazon subsidiary since 1998 (Amazon's first acquisition). For an index builder it is an unusual case: a near-complete bulk copy of the core graph is freely downloadable every day at datasets.imdbws.com, yet its license forbids using that data to "create any kind of online/offline database of movie information." The data is open; the right to index it is not. The legitimate index path is therefore the commercial IMDb Essential Metadata feed sold through AWS Data Exchange at $150K/year (see Verified Metrics), not the free dumps and not crawling — the live site sits behind the same AWS WAF challenge as amazon.com. inferred IMDb monetizes the catalog through licensing and IMDbPro, not the consumer site's ads alone. Basis: the B2B Essential Metadata launch (Sep 2025) and the $150K AWS Marketplace listing both observed this report; the free dataset's no-republish clause channels any indexer toward that paid feed.

Retrieval Notes (site-specific deltas)

Only where IMDb differs from the default platform assumptions.

Identifier & dedup

Two of the web's most reused IDs live here: tt+digits for titles (tt0111161) and nm+digits for people (nm0000138), both printed in the URL (/title/{tt}, /name/{nm}) and adopted as foreign keys by Wikidata, TMDB, and trakt. Dedup against AKA rows (title.akas): one tconst has many regional title strings.

Retrieval unit

The title (tconst) and the person (nconst) are the natural units. Mind the episode explosion: 9.70M of 12.56M titles are tvEpisode (Verified Metrics), so a title index is mostly episodes — group them under their series tconst or they swamp relevance.

Cross-modal joins

Everything joins on tconst/nconst: ratings, crew, principals, episodes, AKAs all key off tconst; principals and crew key off nconst. knownForTitles in name.basics is a person→title edge. These keys are stable and never recycled — an unusually clean join graph.

Authoritative content field

Unlike SEO-stuffed marketplaces, the structured fields are the ground truth here — they are editorially curated, not seller-supplied. originalTitle beats primaryTitle for canonical identity; the AKA table holds the localized display strings.

Freshness & recrawl

The whole core graph is reissued daily as a single dump, so "recrawl" is a diff against yesterday's TSV, not a crawl. Ratings and vote counts move every day; basic catalog facts rarely change after release.

Internal vs external search

On-site /find and /search/* are robots-disallowed (Probe Log), so Google indexes title/name pages but not IMDb's own result pages. The open suggestion endpoint is effectively IMDb's public typeahead and the only sanctioned programmatic lookup.

Indexing Plan

  • Retrieval unit: title (tconst) and person (nconst); episodes rolled up under their parent series; ratings and principals attached as sub-records.
  • Primary keys & joins: tconst and nconst as universal keys; tconst↔nconst via title.principals/title.crew; episode↔series via title.episode; localized strings via title.akas.
  • Ingestion paths, ranked: 1) IMDb Essential Metadata via AWS Data Exchange — the only path licensed for a real index; $150K/yr (Verified Metrics), GraphQL API or bulk. 2) Non-commercial dataset — complete for prototyping/eval only; license bars shipping a database. 3) Suggestion endpoint — typeahead-grade lookups, no bulk. 4) Live crawl — closed by the AWS WAF challenge (Probe Log).
  • Recrawl cadence: daily diff of the seven TSVs for prototypes; for production, the licensed feed's change stream. Ratings daily; catalog facts on change.
  • Derivatives to generate: series-rollup of episodes, person→title bipartite graph from principals, AKA-normalized canonical title, tconst↔Wikidata/TMDB concordance.
  • Demo to build: a daily dump-diff loader — ingest the seven TSVs, count rows by titleType, and emit a JSONL of what changed since yesterday — with a banner stating the non-commercial license forbids republishing the result.

Risks to the Index

  • The free path is a licensing trap. The non-commercial datasets are trivially downloadable but explicitly forbid building any republishable movie database (help center); an index built on them is non-compliant by design. Budget for the commercial feed from day one.
  • inferred Live crawling is structurally closed. Basis: the AWS WAF challenge returned HTTP 202 / 0 bytes to both browser and Googlebot UAs on 2026-06-13 (Probe Log); the SEO sliver is reserved for verified crawler identities.
  • AI crawlers are named and blocked. robots.txt sets Disallow: / for anthropic-ai, Claude-Web, GPTBot, CCbot, Google-Extended, FacebookBot, PiplBot (Probe Log) — AI retrieval over the live site faces enforcement, not licensing.
  • Permission is revocable. IMDb reserves the right to withdraw dataset access at any time and per individual request (conditions of use); a prototype feed is not a durable supply.
