Context

Prime Video is Amazon's video-streaming subsidiary, bundled into Prime membership and carrying ads by default since Jan 29, 2024 unless the subscriber pays for the $4.99/mo Ultra add-on (Verified Metrics). Unlike amazon.com retail, the public site serves full HTML with no AWS WAF JS challenge (Probe Log). Every title carries a stable internal ID, the GTI (amzn1.dv.gti.<uuid>), and X-Ray cast/crew data draws on Amazon-owned IMDb, but no PA-API-equivalent exists for video: the only place catalog metadata (synopsis, cast, genres, maturity rating, audio/subtitle tracks, rating average) appears is inline React SSR JSON on each detail page — undocumented, unversioned, and not exposed as schema.org markup (Probe Log). Reviews route through Amazon's retail ASIN system: each title's review link resolves to an amazon.com/product-reviews/{ASIN} URL, joining the GTI namespace to the retail catalog. For an index builder the interesting object is the title (season/episode GTI), not the page; there is no legitimate bulk path to any modality beyond what a browser renders.

Retrieval Notes (site-specific deltas)

Only where this site differs from the default platform assumptions. Delete any block that would just restate them.

Identifier & dedup

The GTI (amzn1.dv.gti.<uuid>) is Prime Video's ASIN-equivalent: one per season and per episode, embedded in the canonical /detail/{id} URL. Unlike the ASIN, it carries no public documentation or cross-reference table; dedup against title-slug URL variants (/detail/{Title-Slug}/{id} vs bare /detail/{id}, Probe Log) that resolve to the same canonical.

Retrieval unit

Three levels: series → season → episode, each with its own GTI. A search-relevance index should retrieve at the series level but attach season/episode GTIs as sub-records, mirroring the parent-ASIN/child-ASIN split amazon.com uses for product variations.

Cross-modal joins

The one confirmed stable join is GTI→ASIN via the review link (allReviewsLink resolving to amazon.com/product-reviews/{ASIN}, Probe Log) — this is the seam between the video catalog and Amazon's retail review corpus. Cast/director/studio are joined by name string only, no IMDb ID observed in the payload, despite Amazon owning IMDb and using its data to power the in-app X-Ray feature; name-string joins risk collision on common names.

Authoritative content field

The inline SSR JSON is the only content field that exists — there is no separate SEO-facing HTML/schema.org layer to disagree with it (Probe Log: zero JSON-LD, zero microdata). Subtitle tracks marked [auto] (3 of 22 on the sampled title) are machine-translated and should be weighted below human-authored tracks for any transcript-derived field.

Freshness & recrawl

Two regimes, same as amazon.com retail but inverted in what churns fastest: licensed-catalog availability windows change over weeks to months, while ratings/review counts accrue continuously. Live sports and Channels content is the fastest-churning class and carries no stable on-demand GTI once the broadcast ends.

Internal vs external search

/search is robots-disallowed for generic UAs (Probe Log) even though /detail/ pages are crawlable, so a generic crawler can index individual titles but not reconstruct the on-site search/browse graph — title discovery has to come from an external seed list (e.g., IMDb or a licensing database), not from crawling Prime Video's own search.

Indexing Plan

The actionable conclusion: how Site Wise would build the index for this site.

  • Retrieval unit: title (series/film) as the primary document, with season and episode GTIs as fast-refresh sub-records.
  • Primary keys & joins: GTI (amzn1.dv.gti.<uuid>) for the video object; GTI→ASIN via the review-link seam for joining into Amazon's retail review corpus; cast/crew joined by name string only (no ID available).
  • Ingestion paths, ranked: 1) Render /detail/{gti} pages and parse the inline SSR JSON — the only path that yields genres, ratings, audio/subtitle lists; unofficial and schema-fragile. 2) Crawl the generic-UA-allowed /detail/ sliver for SEO-visible title pages (robots.txt permits it; AI agents are blocked). 3) Third-party availability aggregators (e.g., JustWatch-class services) for cross-platform "where to watch" joins, at whatever license those services set — open, not verified here. 4) No SP-API/PA-API equivalent exists for video; there is no seller/developer-account path in.
  • Recrawl cadence: catalog metadata (synopsis, cast, genres) monthly; ratings/review counts weekly; availability windows and live/Channels content daily given fastest churn (Retrieval Notes).
  • Derivatives to generate: ASR transcripts only from human-authored subtitle tracks (skip the 3-of-22 [auto] tracks observed, or flag them low-confidence); poster embeddings for visual dedup across regional key-art variants; a GTI↔ASIN concordance table built by walking the review-link seam.
  • Demo to build: a small crawler that walks a seed list of /detail/{gti} URLs, extracts the inline SSR JSON, and emits a JSONL index doc per title — surfacing exactly which fields (rating histogram, audio/subtitle lists, GTI↔ASIN link) survive a schema change, since none of this is a documented contract.

Risks to the Index

Only risks that change indexing strategy: legal/licensing shifts, API deprecations, ownership changes, wall hardening. Skip generic business risks.

  • No documented contract to build against: every field an indexer would need (genres, ratings, audio/subtitle lists, the GTI↔ASIN seam) comes from an inline SSR JSON blob never intended for external consumption; it can be restructured or minified differently without notice, unlike a versioned API.
  • AI retrieval agents are blocked, not licensed: ~90 named AI/agent user agents get blanket Disallow: / (Probe Log) with no published licensing alternative for video metadata, mirroring the enforcement-not-licensing posture already observed on amazon.com retail.
  • inferred Review-text indexing inherits amazon.com's access risk. Basis: the only path to review bodies for a Prime Video title is its linked amazon.com/product-reviews/{ASIN} page, which sits behind the same retail bot-wall documented in Site Wise's amazon.com report.
  • Pricing/tier churn: the ad-default switch (Jan 2024) and the Ultra rebrand/price hike (Apr 2026, Verified Metrics) show Amazon actively restructuring the paid/ad-free boundary; an indexer tracking "is this ad-free" as a field should expect it to move again.
