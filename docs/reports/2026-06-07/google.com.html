Context

google.com fronts the dominant web index: 5T+ queries a year monetized by the world's largest ad business. For an index builder it is a paradox. Google's product is an index of everyone else's corpus, yet google.com exposes almost nothing of itself: /search is robots-disallowed for every agent, the logged-out SERP is a JavaScript wall (Probe Log), and the one general API — Custom Search JSON — is closed to new customers and dies Jan 1 2027. The rare hard numbers come from a courtroom: US v. Google put the index size (~400B documents) and default-payment economics ($26.3B in 2021) on the record, and the 2025 remedies order created the first compelled access path — index snapshots for "Qualified Competitors". inferred Google polices its SERP boundary with engineering rather than crawler bans. Basis: robots.txt observed 2026-06-10 has zero AI-crawler user-agent rules — /search is simply disallowed for all and the SERP requires JS (Probe Log).

Retrieval Notes (site-specific deltas)

Only where google.com differs from the default platform assumptions.

The inversion

Other reports ask how to get a site's corpus out. Google's corpus is the open web — available to anyone willing to crawl it. What is locked is the derived layer: ranking, query logs, click data — exactly what the remedies order targets.

Identifier & dedup

google.com URLs are not stable objects: SERP URLs encode session params, and country editions collapsed onto google.com in 2017. The durable IDs are the query string and Knowledge Graph MIDs (/m/…, /g/…), which join KG API responses, Trends topics, and knowledge panels.

Retrieval unit

A query→composition pair, not a page: ranked links, ads, AI Overview, and vertical modules assembled per request, non-deterministic by design. Any archived SERP is one sample, not the canonical answer.

Authoritative content field

AI Overviews replace the snippet as the page's voice: the authoritative text is generated, sources demoted to citations. Answer text and cited URLs need separate fields with separate trust.

Quality & spam

The adversarial pressure runs through google.com rather than at it: SEO manipulates Google's index of other sites. google.com's own pages (brochureware, per the sitemap census) are clean and tiny.

Internal vs external search

The site: escape hatch terminates here — no external engine searches Google's results, and SERPs are deliberately unindexable (robots + JS wall, Probe Log).

User query profile

The full distribution of human intent, 5T+/year (Verified Metrics) — the dataset itself. The suggest endpoint leaks its head, one prefix at a time; the tail is visible only inside Google and in per-owner Search Console slices.

Freshness & recrawl

Nothing on google.com is worth scheduled recrawl except suggestions and Trends; everything else either never changes (marketing pages) or changes every request (SERPs).

Indexing Plan

  • Retrieval unit: the query — with attached suggestion sets, Trends series, KG entities, and (where capturable) answer compositions.
  • Primary keys & joins: normalized query string + language + region; KG MID for entity joins; source URL for AI Overview citations.
  • Ingestion paths, ranked: 1) Suggest endpoint — free, unauthenticated prefix-fanout for query distribution (fragile: unofficial). 2) KG Search API + Trends — entity and demand signals, key-gated. 3) Search Console — ground-truth query→click data, own properties only. 4) CSJA — legacy accounts only, dead Jan 1 2027. 5) Qualified Competitor snapshot — the only bulk path to the index itself, court-gated. 6) SERP scraping — no legitimate basis; excluded.
  • Recrawl cadence: suggestions daily on a tracked prefix set; Trends weekly; KG entities monthly; marketing pages never.
  • Derivatives to generate: query-intent clusters from suggestion fanouts; entity-linking table (query→MID); a divergence log of identical queries sampled across regions to quantify non-determinism.
  • Demo to build: a suggest-fanout crawler — seed term → recursive prefix expansion (a–z appended) → ranked query graph with subtype codes, demonstrating how much query-distribution signal the one open endpoint leaks.

Risks to the Index

  • The last general API has a death date: Custom Search JSON API is closed to new customers and returns errors after Jan 1 2027 (docs); Google's pointer is Vertex AI Search, which searches your corpus, not the web.
  • The compelled path is contested: Google appealed the antitrust ruling in May 2026 (report); Qualified Competitor data access could be narrowed or delayed on appeal.
  • The suggest endpoint is a courtesy, not a contract: undocumented, unauthenticated (Probe Log) — it can be keyed, quota'd, or closed without notice, taking the only open ingestion path with it.
  • inferred SERP capture will keep getting harder. Basis: the logged-out SERP already requires JS execution even for a browser UA (Probe Log, 2026-06-10), a hardening Google shipped without robots.txt changes — enforcement is moving into the client runtime where third parties can't negotiate.
