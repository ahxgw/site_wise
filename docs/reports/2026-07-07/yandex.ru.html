yandex.ru

yandex.ru is Russia's dominant search portal (~72% share), but its homepage has redirected to VK-owned dzen.ru since 2022, splitting ownership across one domain.

Context

yandex.ru launched Russia's first major search engine in 1997 and today runs under MKAO Yandex (Moscow Exchange: YDEX), a Russian holding company that Yandex N.V. sold its Russian operations into for ₽475B in July 2024 — a transaction separate from, and two years after, a smaller divestiture that still shapes this exact host. In August–September 2022 Yandex sold its News and Zen products, and the yandex.ru homepage itself, to VK (state-controlled) in exchange for the Delivery Club food-delivery business. Since then the root path https://yandex.ru/ 302-redirects to dzen.ru, a VK property, while dozens of Yandex product paths — Maps, Images, Realty, Support, Weather, Q&A — keep running on the same yandex.ru domain and cookie space. inferred The crawl/ownership context of this host is not uniform: the root belongs to VK's corpus, the paths belong to Yandex's. Basis: the root's x-robots-tag: unavailable_after: 12 Sep 2022 header matches the publicly reported completion date of the VK sale (Source Notes). Yandex itself now treats ya.ru, not yandex.ru, as its primary logged-in entry point. Business model on the Yandex side: search/display advertising plus e-commerce, ride-hailing and delivery commissions, and Yandex Plus subscriptions.

Retrieval Notes (site-specific deltas)

Only where this host differs from the default platform assumptions — mostly because it isn't one product.

Identifier & dedup

The unit that matters here is the URL path prefix, not the host. yandex.ru/ and yandex.ru/images/ belong to different owners with different crawl policies (Context); an indexer must dedup and attribute by prefix, not by domain.

Retrieval unit

No single unit spans the host. Per vertical: a help article, a Q&A thread, an image-result page, a realty listing. Treat each vertical's sitemap as its own corpus.

Cross-modal joins

open — no shared entity ID was observed linking Images, Q&A, Realty, and Support sitemaps to each other; each vertical appears to run its own namespace.

Authoritative content field

For the Q&A archive, the existing HTML is authoritative since the product is being sunset toward Alice (no further edits expected). For Uslugi, the live uslugi.yandex.ru subdomain should outrank any cached yandex.ru/uslugi alias.

Freshness & recrawl

Split by product lifecycle rather than a host-wide rate: sunset verticals (Q&A) need rare recrawl; active verticals (Realty, Images, Support/blog) follow ordinary weekly-to-monthly cadences.

Internal vs external search

The on-site /search endpoint is unusable for automated retrieval — every plain-HTTP request hit an interactive CAPTCHA in this probe, on both yandex.ru and yandex.com. Build against sitemaps and the paid Search API; never against live SERP scraping.

Indexing Plan

  • Retrieval unit: vertical-scoped page (help/blog article, Q&A thread, realty listing, image-result page) — never the whole host as one corpus.
  • Primary keys & joins: URL path-prefix as pseudo-namespace per vertical; no shared cross-vertical entity ID found (Retrieval Notes, open).
  • Ingestion paths, ranked: 1) Sitemap crawl of the 24 Sitemap: directives — free, no auth, covers support/blog/Q&A/images/video/realty/finance/medicine/uslugi. 2) Yandex Search API — paid, PAYG billing, for live SERP text at scale. 3) Yandex Maps API — paid tiers, for geo/business entities. 4) Public crawl of /search results — closed; CAPTCHA-gated even for plain requests (Probe Log). Do not design around it.
  • Recrawl cadence: help/blog weekly; Q&A archive cold (sunset product); Realty/Uslugi listings on a churn rate that is currently open; Images sitemap index re-walked monthly to catch new shard numbers.
  • Derivatives to generate: Russian→target-language translation of help/blog content if indexing for non-Russian users; no OCR/ASR need is evident from this probe (open — verticals not deep-sampled).
  • Demo to build: a small "domain ownership mapper" — walk each top-level path with a browser UA, classify the response (200 direct, self-canonical redirect, off-host redirect, redirect into Passport/CAPTCHA), and render the split-ownership map of yandex.ru observed in this Probe Log.

Risks to the Index

  • The host is not one policy: the root inherited VK's corpus and crawl context in 2022 while subpaths remain Yandex's; indexing "yandex.ru" as a single owner will misattribute content (Context, Probe Log).
  • SERP access is walled, not just discouraged: every plain-HTTP request to /search hit an interactive CAPTCHA in this probe on both yandex.ru and yandex.com; any plan assuming crawlable search results will silently fail.
  • inferred Geo-targeting can move a vertical off this host entirely. Basis: /maps/ 302-redirected to yandex.eu/maps/ from this probe's non-Russian vantage point — a crawler seeded on yandex.ru alone could silently lose Maps coverage depending on request origin.
  • Ownership sits on Russia-domiciled, sanctions-era infrastructure: MKAO Yandex was restructured out of a Dutch parent in 2024 under a discounted forced sale, and the root path already changed hands once (2022, to a state-controlled company); further ownership or access changes on short notice are plausible.
