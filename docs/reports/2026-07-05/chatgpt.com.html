Context

ChatGPT is OpenAI's conversational AI product, launched 2022-11-30 as a free research preview on GPT-3.5. OpenAI was founded 2015-12-11 and completed a restructure on 2025-10-28 into OpenAI Group PBC, a public-benefit for-profit controlled by the nonprofit OpenAI Foundation (26% stake); CEO is Sam Altman. The business model on this domain is freemium subscriptions (Free / Go / Plus / Business / Enterprise / Edu); the underlying models are separately monetized as a metered token API on platform.openai.com/api.openai.com, a distinct product not covered here. For an index builder, chatgpt.com is close to the anti-corpus case in a different way than a tool site: it isn't that content is deleted, it's that almost nothing on the domain is fetchable at all without passing a Cloudflare Managed Challenge (see Probe Log), and what little is allowlisted in robots.txt is static marketing copy, not user content. inferred The allowlisted paths in robots.txt are chatgpt.com's entire legitimately crawlable surface. Basis: the Disallow: / catch-all after the allow list (Probe Log), and a site:chatgpt.com search returning only those same paths (/pricing/, /g/*, /atlas/, /codex/pricing/).

Retrieval Notes (site-specific deltas)

Transport gate before content gate

The default-platform assumption is a curated SEO sliver behind a UA/robots split; here that sliver additionally sits behind a Cloudflare Managed Challenge that blocks non-verified-bot UAs from every path, including the allowed ones (Probe Log). A crawler needs verified-bot status, not just an allowed path.

Identifier & dedup

chat.openai.com is a pure 308 alias to chatgpt.com (Probe Log) — treat as one canonical host. /share/<uuid> tokens are the only stable per-object IDs reachable without auth, and they're revocable.

Authoritative content field

For GPT/App storefront pages, the listing description is the only text OpenAI publishes; the underlying prompt/instructions are not exposed.

Indexing Plan

The actionable conclusion: how Site Wise would build the index for this site.

  • Retrieval unit: individual marketing/landing page for the static sliver; individual /g/<slug> listing for the GPT/App directory. No unit exists for conversations or shared links at bulk scale.
  • Primary keys & joins: canonical URL path; GPT/App slug from its /g/ URL. No cross-modal joins exist since there is no image/video/audio corpus on this host.
  • Ingestion paths, ranked: 1) rely on a general web index that already holds a verified-bot allowlisting with Cloudflare (coverage ceiling: exactly the ~90 robots-allowed paths); 2) render with a real browser to pass the Managed Challenge for the same path set (higher cost, same ceiling); 3) no bulk path exists for conversations, GPTs' underlying prompts, or the Image Library — don't budget for them.
  • Recrawl cadence: marketing pages, weekly (tied to product launches); GPT/App directory, monthly (listing churn is slow relative to the underlying product).
  • Derivatives to generate: none justified — no media corpus exists to run OCR/ASR/embeddings against beyond ordinary page-text embeddings.
  • Demo to build: a small crawler that fetches the five sitemap targets from a real browser context (to clear the Managed Challenge) and diffs the resulting URL set against the robots.txt allow-list, to catch newly launched landing pages (Atlas, football, shopping-style feature pages) as an early product-launch signal.

Risks to the Index

  • Cloudflare Managed Challenge blocks any non-verified-bot crawler outright; an index that isn't on Cloudflare's verified-bot list gets zero coverage of even the allowed paths, not degraded coverage.
  • The Aug 2025 shared-link indexing incident shows OpenAI actively removes previously-crawlable object classes on privacy grounds — any shared-conversation coverage built today could vanish or become a takedown liability.
  • GPTs and the newer Apps directory are two overlapping, actively-evolving object types (GPT Store retained alongside the Dec 2025 Apps SDK rollout); an index built against one schema risks missing the other.
  • Sora's shutdown (2026-04-26 app, 2026-09-24 API) shows OpenAI will fully discontinue a product surface with data deletion and no export guarantee — any indexing plan for an OpenAI-family host should assume host-level mortality, not just content churn.
