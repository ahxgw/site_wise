Context

Facebook launched in February 2004, IPO'd in 2012, and since October 2021 has been one app inside Meta Platforms, which sells advertising against the combined Family of Apps. Meta stopped reporting Facebook-only users after FY2023; the last clean anchor is 3.07B MAU (Dec 2023). For an index builder, Facebook is the canonical closure case: a corpus that was once partially crawlable and API-readable has been walled off in stages — the 2018 post-Cambridge-Analytica API lockdown, the 2020 oEmbed token requirement, and the August 2024 CrowdTangle shutdown, replaced by a researcher-only Content Library. What remains public-by-design is a thin, Meta-curated SEO surface: sitemaps of public group posts, high-follower profiles, and events, served only to approved search crawlers (the same files return 403 to everyone else, Probe Log). inferred Meta actively tunes which public group posts to expose to search engines rather than exposing the public corpus wholesale. Basis: the 148 sitemap pointers observed in robots.txt carry experiment-style names like groups_llm_bf_posts, groups_target_boosting_with_firehose_posts, and groups_query_boost_tpc_posts, i.e. ranked/selected shards, not an exhaustive census.

Retrieval Notes (site-specific deltas)

Only where Facebook differs from the default platform assumptions.

Identifier & dedup

Numeric object IDs (page, group, post, event) are stable, but post permalinks now use opaque pfbid tokens and one object has many URL skins (www, m, mbasic, web, vanity names, story.php params). Canonicalize to the numeric ID whenever an API surface exposes it; URL-string dedup alone will overcount.

Retrieval unit

The post alone is not the unit — meaning and visibility live in the wrapper: actor, page/group context, audience setting, and the comment thread. MCL models this correctly (posts carry their surface and follower-tier context); any index should too.

Freshness & recrawl

Deletion has three flavors here: actual deletion, privacy-state change, and group setting change — the last two silently de-publish content an index already holds. Re-verification matters more than recrawl-for-new on this site.

Authoritative content field

Link-share posts carry Open Graph preview text scraped from the target at share time; it goes stale independently of both the post and the target page. Treat preview text as a cached third-party claim, not content.

Internal vs external search

The public slice Google can see is Meta-selected (the experiment-named sitemap shards in the Probe Log), so site:facebook.com coverage reflects Meta's SEO strategy, not the public corpus. Internal search is the only complete index, and it is personalized and logged-in only.

Quality & spam

The objects most reachable from outside (public group posts, Marketplace-adjacent pages) skew toward the spammiest tier — engagement-bait groups, resharing rings, scam listings — while the highest-trust content (friend graph, private groups) is unreachable by design. External samples are adversely selected.

User query profile

People, local businesses, groups, events, second-hand goods. These resolve through the social graph (who you know, where you are) — exactly the join an external index cannot make, which caps the usefulness of any logged-out Facebook index regardless of coverage.

Indexing Plan

  • Retrieval unit: page/group/event entity as the primary document; individual public posts only as attachments to an entity, with surface and audience context.
  • Primary keys & joins: numeric object ID; entity → posts → comments by parent ID; place pages join to maps/local corpora by name + address.
  • Ingestion paths, ranked: 1) Pages API under PPCA — page metadata and posts, app review + business verification, 4,800 × engaged-users daily quota; the only commercial-grade path. 2) oEmbed (app token, post review) — display-only rendering of individual public posts/videos, no bulk reads. 3) Meta Content Library — the widest corpus (public groups included) but not-for-profit research only; unusable for a commercial index, useful for calibration studies. 4) Crawling — closed: default-deny robots.txt, login walls, and 403'd sitemaps (Probe Log).
  • Recrawl cadence: page/place entities monthly; page posts via API webhooks where granted; re-verify held URLs quarterly for de-publication rather than crawling for new content.
  • Derivatives to generate: entity resolution between Facebook pages and external business records (the durable value is the place/business directory, not the feed); language ID on public group posts if MCL-calibrated sampling is ever available.
  • Demo to build: a robots.txt diff-tracker that snapshots Facebook's 148 sitemap pointers and named-agent rules daily — the experiment-style shard names visibly change, and the churn is a direct read on Meta's SEO-exposure strategy.

Risks to the Index

  • Access only narrows: 2018 API lockdown → 2020 oEmbed token gate → Aug 14 2024 CrowdTangle shutdown (Meta). Each step removed a working ingestion path with months of notice. Plan for PPCA or oEmbed to be next.
  • The research path is regulator-dependent: Meta describes MCL as built to "meet new regulatory requirements for data-sharing" (same page) — i.e. DSA-driven. If the regulatory pressure changes, so may the tool.
  • SEO surface is an experiment, not a commitment: the sitemap shards carry test names and can be withdrawn silently; any pipeline keyed to what Google sees of Facebook inherits Meta's A/B schedule.
  • inferred Anti-scraping enforcement is contractual and technical, not just robots.txt. Basis: the robots.txt header asserts that automated collection requires express written permission per Meta's Automated Data Collection Terms, and the Probe Log shows even logged-out page views rejected (HTTP 400 / login 302) for non-browser clients.
