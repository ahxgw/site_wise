Context

YouTube is Alphabet's user-generated video platform: 20B+ uploaded videos monetized through ads and a subscription stack (Premium, Music, TV, NFL Sunday Ticket), with creators paid a revenue share that exceeded $100B over four years. For an index builder it is the densest audiovisual corpus on the web, keyed by one of the web's best identifiers — the 11-character videoId — and unusually open at the page level: robots.txt bans no AI crawlers and watch pages serve full JSON-LD to logged-out clients (Probe Log). The openness stops exactly where the payload starts. inferred YouTube's crawl posture is calibrated to feed search engines page by page while preventing anyone from reconstructing the catalog or the media. Basis: robots.txt allows /watch but disallows /results, /feeds/videos.xml, and /youtubei/; both sitemap indexes contain zero watch URLs; caption endpoints return empty bodies to non-browser clients (all observed 2026-06-10, Probe Log).

Retrieval Notes (site-specific deltas)

Only where YouTube differs from the default platform assumptions.

Identifier & dedup

The 11-char videoId is printed in every surface — page URL, oEmbed, RSS, thumbnail URL, embed. Collapse /watch?v=, youtu.be/, /shorts/, /embed/, /live/ onto it. Channels have two names: channelId (UC…, stable) and @handle (mutable) — key on the former.

Retrieval unit

The ideal unit — transcript segment + timestamp — is unavailable to third parties (Access Matrix). The practical unit is the video record (metadata, thumbnail, stats) with the channel as its trust ancestor; accept that the answer payload stays behind the player.

Cross-modal joins

Everything joins on videoId, and the thumbnail join costs zero API calls — the URL is derivable from the ID alone (Probe Log). Caption tracks key on videoId + language; auto-dubbing means audio language no longer identifies a distinct object.

Authoritative content field

Titles and thumbnails are CTR-optimized bait, the least trustworthy fields on the site. Descriptions mix summary with sponsor links and SEO tags. What the video actually says lives in captions — which are exactly what is gated.

Freshness & recrawl

No firehose and no archive feed: per-channel RSS exposes only the latest 15 items, and API discovery is capped at 100 search.list calls/day. Freshness at 20M uploads/day is only achievable channel-by-channel on a tracked set.

Quality & spam

Trust attaches to channels, not videos. Site-specific manipulation: reupload/compilation farms, AI-generated slop at upload-rate scale, clickbait thumbnail A/B-testing, and engagement-bait Shorts recut from long-form.

Internal vs external search

/results is robots-disallowed, so no external engine can mirror YouTube's SERP. Google Search carries external recall for its sister company — a path no other index gets. site:youtube.com queries are common precisely because on-site search ranks by engagement, not relevance alone.

User query profile

Procedural ("how to…"), music, entertainment, news clips, reviews. The answers are inside time-based media, so a metadata-only index can rank videos but cannot answer from them — it routes, it doesn't retrieve.

Indexing Plan

  • Retrieval unit: video record keyed by videoId (metadata + thumbnail + stats + channel), upgraded to transcript segments only if a licensed path appears.
  • Primary keys & joins: videoId; channelId for trust and freshness; videoId+language for caption/audio tracks.
  • Ingestion paths, ranked: 1) Data API videos.list — key-only, ~500K videos/day hydrated per project (Verified Metrics); hydrates but cannot discover. 2) Watch-page JSON-LD crawl — robots-permitted, quota-free, rich (Probe Log), but seed discovery must come from external links since no sitemap or search surface exists. 3) Channel RSS for freshness on a tracked channel set — unauthenticated but robots-disallowed, so treat as gray and prefer API playlistItems.list on upload playlists. 4) oEmbed for display cards. 5) Transcripts/media: no legitimate path — design the index to route to the player, not replace it.
  • Recrawl cadence: tracked channels hourly (uploads); head-video stats daily; tail metadata monthly; thumbnails on change detection via ETag.
  • Derivatives to generate: thumbnail embeddings (visual dedup of reupload farms), entity extraction from titles/descriptions. No ASR — there is no legitimate audio access to feed it.
  • Demo to build: a quota-budget planner — given N seed channels, simulate the playlistItems + videos.list schedule and show coverage and staleness against the 10K-unit/day budget.

Risks to the Index

  • API discovery already shrank: the default quota now caps search.list at 100 calls/day outright (docs, Jun 2026) — older plans assumed 100 units/call against 10K units. Quota terms move without notice; hydration could be next.
  • The open robots.txt can flip: the entire quota-free JSON-LD path rests on a ~30-line robots file with no AI rules (Probe Log). A default-deny change closes it overnight, as peers already have.
  • Training access is being formalized as opt-in licensing: the third-party training program (off by default) channels AI access into deals YouTube brokers — strengthening, not weakening, the case against unlicensed media use.
  • inferred Legal exposure exceeds technical exposure. Basis: ToS prohibits scraping and downloading even on paths robots.txt permits, while caption endpoints silently serve empty bodies (Probe Log) — access that works today is not access that is licensed.
