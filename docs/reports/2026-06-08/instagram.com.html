Context

Instagram is Meta's image-and-short-video network, bought for about $1B in 2012 and now at 3B monthly users; Meta otherwise blends it into Family of Apps reporting and monetizes it through the ad auction. For an index builder it is the web's largest visual corpus and also its most sealed: the consumer API line ended with the Basic Display API shutdown (Dec 2024), oEmbed returns an embed shell without author or thumbnail, and the Graph API reaches only professional accounts that grant your app access. What remains public is a thin SEO surface — profile and post pages whose Open Graph tags are served to non-browser agents while real browser UAs get an empty JS shell (Probe Log). The one bulk path, Meta Content Library, is gated to nonprofit researchers. inferred The narrow SEO surface is intentional, not incidental. Basis: robots.txt ships six sitemap pointers limited to profiles, places, and keyword pages, while Googlebot and Bingbot are specifically disallowed from /*/comments/ and /*/liked_by/ (observed 2026-06-10).

Retrieval Notes (site-specific deltas)

Only where Instagram differs from the default platform assumptions.

Identifier & dedup

The 11-character shortcode (/p/{shortcode}/, /reel/{shortcode}/) is the public media ID and survives caption edits. Graph API uses a separate numeric media ID; usernames are mutable while the underlying user ID is stable — keep both. CDN URLs are useless as keys: signed params rotate per fetch.

Retrieval unit

Two units: the media object (image/Reel + caption + creator) for content queries, and the profile entity for the dominant navigational queries. Stories never become retrieval units — they expire in 24h and have no logged-out surface at all.

Cross-modal joins

Inside the API, media ID joins caption, comments, audio track, location, and tagged users. Outside it, the only join surface is the OG tag set — shortcode ↔ image ↔ caption excerpt ↔ username — with no numeric IDs exposed.

Authoritative content field

The caption is the only author-written text; everything else (OG title/description) is machine-generated from it plus profile stats. Visual/ASR understanding has to carry most of the relevance signal because captions are short, hashtag-stuffed, and often unrelated to the image.

Freshness & recrawl

Three regimes: Stories (gone in 24h, skip), Reels/posts (relevance decays in days, object persists), profile stats (continuous churn, cheap via OG tags). Deletion and private-toggling are unsignaled — expect silent corpus shrinkage.

Internal vs external search

Instagram search is entity search — accounts, hashtags, places, audio — not full-text. The logged-out keyword pages in the sitemap set (dim_ig_logged_out_web_keyword_ent, Probe Log) show Meta building a parallel SEO-only keyword surface rather than exposing real search.

User query profile

Heavily navigational (person, brand, creator) plus visual discovery (style, place, recipe). For a web index, the profile entity answers most Instagram-bound queries; individual post retrieval matters mainly for news/meme provenance.

Indexing Plan

  • Retrieval unit: profile entity (primary) and media object keyed by shortcode (secondary).
  • Primary keys & joins: user ID + username history for profiles; shortcode for media; never CDN URLs.
  • Ingestion paths, ranked: 1) Profile/post OG tags served to non-browser agents — free, logged out, yields stats + caption excerpt, but unofficial and revocable. 2) Graph API + Business Discovery — deep fields, only for professional accounts that opt in. 3) Meta Content Library — the only bulk path, closed to commercial builders. 4) oEmbed — display-only shell carrying no metadata (Probe Log).
  • Recrawl cadence: profile OG stats weekly for head accounts; posts on-demand at query time; skip Stories entirely.
  • Derivatives to generate: image embeddings and OCR only for media with a license path (owned/consented accounts); username-change history table for entity continuity.
  • Demo to build: an OG-surface profiler — paste usernames, fetch with a non-browser UA, parse follower/following/post counts and caption excerpts into JSONL, and log when Meta turns the surface off.

Risks to the Index

  • The consumer API line is closed and still closing: Basic Display API died Dec 4 2024 (Meta); oEmbed lost author/thumbnail fields (Probe Log). Plan for zero third-party content API, permanently.
  • The OG ingestion path is unofficial: it depends on UA-conditional server rendering Meta can flip off silently, and bulk use conflicts with the robots.txt written-permission banner — treat it as best-effort enrichment, not a foundation.
  • MCL access is conditional: nonprofit-only eligibility excludes a commercial index, and enclave terms bar exporting raw data; a strategy assuming MCL coverage has no fallback.
  • inferred Enforcement posture is contractual as well as technical. Basis: robots.txt opens with a written-permission requirement binding "all authorized user-agents" to Meta's Automated Data Collection Terms (observed 2026-06-10) — crawl tolerance is framed as a revocable license, not an open norm.

Inferences

  • The narrow SEO surface is deliberate product policy. Basis: robots.txt sitemap set covers only profiles/places/keywords while Googlebot is excluded from comments and likes; OG tags are served to non-browser agents but stripped from the browser-UA render (Probe Log, 2026-06-10).
  • CDN media cannot be hotlink-archived. Basis: observed og:image URLs carry signed, expiring parameters (oh=, oe=, _nc_*) on scontent-*.cdninstagram.com.
  • Crawl tolerance is a revocable license, not a norm. Basis: the robots.txt written-permission banner binding authorized agents to Meta's Automated Data Collection Terms.
